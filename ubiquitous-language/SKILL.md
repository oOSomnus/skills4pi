# 通用语言

从当前对话中提取并规范化领域术语，形成一致的词汇表，保存到本地文件。

## 流程

1. **扫描对话**，查找与领域相关的名词、动词和概念
2. **识别问题**：
   - 同一个词用于不同的概念（歧义）
   - 不同的词用于同一个概念（同义词）
   - 模糊或含混的术语
3. **提出规范词汇表**，做出有主见的术语选择
4. **写入 `UBIQUITOUS_LANGUAGE.md`** 到工作目录，使用以下格式
5. **在对话中输出摘要**

## 输出格式

写入 `UBIQUITOUS_LANGUAGE.md` 文件，结构如下：

```md
# 通用语言

## 订单生命周期

| 术语          | 定义                                       | 应避免的别名            |
| ------------- | ------------------------------------------ | ----------------------- |
| **Order**     | 客户购买一件或多件商品的请求               | Purchase, transaction   |
| **Invoice**   | 发货后发送给客户的付款请求                 | Bill, payment request   |

## 人员

| 术语           | 定义                                 | 应避免的别名              |
| -------------- | ------------------------------------ | ------------------------- |
| **Customer**   | 下订单的个人或组织                   | Client, buyer, account    |
| **User**       | 系统中的身份验证身份                 | Login, account            |

## 关系

- 一个 **Invoice** 属于恰好一个 **Customer**
- 一个 **Order** 生成一个或多个 **Invoice**

## 示例对话

> **开发者：** "当 **Customer** 下 **Order** 时，我们会立即创建 **Invoice** 吗？"
> **领域专家：** "不会——**Invoice** 只在 **Fulfillment** 确认后才生成。一个单独的 **Order** 如果商品以单独的 **Shipment** 发货，可以生成多个 **Invoice**。"
> **开发者：** "所以如果 **Shipment** 在发货前被取消，就没有 **Invoice** 存在？"
> **领域专家：** "完全正确。**Invoice** 的生命周期与 **Fulfillment** 绑定，而不是 **Order**。"

## 标记的歧义

- "account" 曾被用来同时表示 **Customer** 和 **User**——这是两个不同的概念：**Customer** 下订单，而 **User** 是可能不代表 **Customer** 的身份验证身份。
```

## 规则

- **要有主见。**当同一个概念有多个词时，选择最好的一个，把其他的列为应避免的别名。
- **明确标记冲突。**如果某个术语在对话中被含混使用，在"标记的歧义"部分明确指出，并给出清晰的建议。
- **只包含与领域专家相关的术语。**跳过模块或类的名称，除非它们在领域语言中有意义。
- **保持定义紧凑。**最多一句话。定义它是什么，而不是它做什么。
- **展示关系。**使用粗体术语名称，并在明显时表达基数关系。
- **只包含领域术语。**跳过通用编程概念（数组、函数、端点），除非它们有领域特定的含义。
- **当存在自然聚类时，将术语分组到多个表格中。**（例如按子域、生命周期或角色）。每个组获得自己的标题和表格。如果所有术语属于一个单一的内聚领域，一个表格也可以——不要强制分组。
- **编写示例对话。**开发者与领域专家之间的简短对话（3-5 轮交流），展示术语如何自然交互。对话应澄清相关概念之间的边界，并展示术语如何被精确使用。

## 示例对话

> **开发者：** "如何在没有 Docker 的情况下测试 **sync service**？"

> **领域专家：** "提供 **filesystem layer** 而不是 **Docker layer**。它实现相同的 **Sandbox service** 接口，但使用本地目录作为 **sandbox**。"

> **开发者：** "所以 **sync-in** 仍然创建 **bundle** 然后解包？"

> **领域专家：** "完全正确。**sync service** 不知道它正在与哪个层通信。它调用 `exec` 和 `copyIn`——**filesystem layer** 只是将它们作为本地 shell 命令运行。"

</example>

## 重新运行

在同一对话中再次调用时：

1. 读取现有的 `UBIQUITOUS_LANGUAGE.md`
2. 纳入后续讨论中的任何新术语
3. 如果理解有演进，更新定义
4. 重新标记任何新的歧义
5. 重写示例对话以纳入新术语

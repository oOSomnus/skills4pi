# 深模块

源自《软件设计的哲学》：

**深模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│   小接口              │  ← 少量方法，简单参数
├─────────────────────┤
│                     │
│                     │
│   深实现              │  ← 隐藏的复杂逻辑
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（避免）

```
┌─────────────────────────────────┐
│       大接口                      │  ← 许多方法，复杂参数
├─────────────────────────────────┤
│  薄实现                           │  ← 仅仅传递
└─────────────────────────────────┘
```

设计接口时，问：

- 我能减少方法的数量吗？
- 我能简化参数吗？
- 我能在内部隐藏更多的复杂性吗？

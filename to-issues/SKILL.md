# 转换为任务

使用垂直切片（追踪子弹）将计划拆分为可独立完成的任务，写入项目 `.pi/` 目录下的文件中。

## 流程

### 1. 收集上下文

从对话上下文中已有的内容开始工作。如果用户传递了参数，将其作为上下文使用。

### 2. 探索代码库（可选）

如果你尚未探索代码库，可以探索以了解代码的当前状态。

### 3. 起草垂直切片

将计划拆分为**追踪子弹**任务。每个任务是一个薄的垂直切片，端到端地穿越所有集成层，而不是一个层的水平切片。

切片可以是 'HITL' 或 'AFK'。HITL 切片需要人为交互，如架构决策或设计审查。AFK 切片可以在不需要人为交互的情况下实现和合并。尽可能优先选择 AFK。

<vertical-slice-rules>
- 每个切片提供一条穿越每个层（模式、API、UI、测试）的窄但完整的路径
- 一个完成的切片本身就可以演示或验证
- 优先选择多个薄切片，而不是少数厚切片
</vertical-slice-rules>

### 4. 询问用户

将提议的拆分呈现为一个编号列表。对每个切片，展示：

- **标题**：简短的描述性名称
- **类型**：HITL / AFK
- **被阻塞于**：哪些其他切片（如果有的话）必须先完成
- **覆盖的用户故事**：此切片涵盖哪些用户故事（如果源材料有的话）

询问用户：

- 粒度感觉合适吗？（太粗 / 太细）
- 依赖关系正确吗？
- 任何切片应该合并或进一步拆分吗？
- 正确的切片被标记为 HITL 和 AFK 了吗？

迭代直到用户批准拆分。

### 5. 创建任务文件

对每个批准的切片，将其写入项目 `.pi/` 目录下的独立文件中。

文件命名：`.pi/task-{编号}-{简短描述}.md`

按依赖顺序创建（阻塞者优先），以便可以在"被阻塞于"字段中引用真实的文件路径。

文件模板：
```markdown
## 父级

{父文件路径}（如果来源是另一个文件，否则省略此部分）

## 要构建什么

对此垂直切片的简洁描述。描述端到端的行为，而不是逐层的实现。

## 验收标准

- [ ] 标准1
- [ ] 标准2
- [ ] 标准3

## 被阻塞于

- 被阻塞于 {文件路径}（如果有的话）

或"无——可以立即开始"（如果没有阻塞）。
```

不要关闭或修改任何父任务文件。
